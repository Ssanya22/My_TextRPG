# UI для текстовой RPG в Unity — пошаговая инструкция

## Требования

- Unity с установленным **TextMeshPro** (при первом добавлении TMP-элемента Unity предложит импорт — нажмите «Import TMP Essentials»).
- Сцена с камерой (по умолчанию есть в новой сцене).

---

## 1. Canvas и настройка разрешения/масштабирования

### 1.1. Создание Canvas

**Что такое Canvas?** **Canvas** — это специальный объект в Unity, который является "холстом" для всех UI-элементов (кнопки, текст, поля ввода и т.д.). Все UI должно быть внутри Canvas, чтобы правильно отображаться на экране.

1. В панели **Hierarchy** (слева в Unity, где список всех объектов сцены) правый клик → **UI** → **Canvas**.
2. Unity создаст:
   - **Canvas** (корень UI);
   - **EventSystem** (если его ещё не было) — нужен для обработки кликов и ввода.

### 1.2. Настройка Canvas

1. Выберите объект **Canvas** в **Hierarchy** (панель слева).
2. В панели **Inspector** (справа в Unity, где отображаются свойства выбранного объекта) у компонента **Canvas**:
   - **Render Mode**: **Screen Space - Overlay** (UI поверх всего).
   - **Pixel Perfect** (по желанию): включите для чёткой отрисовки пикселей.

3. У компонента **Canvas Scaler**:
   - **UI Scale Mode**: **Scale With Screen Size**.
   - **Reference Resolution**: например **1920 x 1080** (или 1280x720).
   - **Screen Match Mode**: **Match Width Or Height**.
   - **Match**: **0.5** (баланс между шириной и высотой; сдвиг в сторону **1** — больше подстраивается под высоту).

Так UI будет масштабироваться под разное разрешение и соотношение сторон.

### 1.3. Дополнительно (по желанию)

- **Reference Pixels Per Unit**: оставьте 100 или подстройте под спрайты.
- Для **разных соотношений сторон** можно менять **Match** (0 — по ширине, 1 — по высоте).

## 2. Большое поле TextMeshPro с прокруткой

### 2.1. Панель под текст

1. Правый клик по **Canvas** → **UI** → **Panel** (или **Create Empty** и добавьте **Image** как фон).
2. Назовите объект, например **Panel_Log**.
3. Настройте **RectTransform**:
   - **Anchor**: левый верхний угол (Top-Left): Min (0, 1), Max (0, 1).
   - **Pivot**: (0, 1).
   - **Pos X**: 10, **Pos Y**: -10, **Width**: 700, **Height**: 400 (или под ваше разрешение).
   - **Left / Top**: 10, **Right**: оставьте 0 или подгоните под панель статистики справа.

### 2.2. Scroll View

1. Правый клик по **Panel_Log** → **UI** → **Scroll View**.
2. Unity создаст:
   - **Scroll View** (с **Scroll Rect**, **Image**, **Mask**);
   - внутри: **Viewport** → **Content**.

Настройте **Scroll Rect** у **Scroll View**:

- **Content**: перетащите сюда объект **Content** (дочерний к Viewport).
- **Horizontal**: снять галочку (только вертикальная прокрутка).
- **Vertical**: оставить галочку.
- **Movement Type**: **Elastic** или **Clamped**.
- **Scroll Sensitivity**: 20–30.

### 2.3. Текст (TextMeshPro) внутри Scroll View

**Что такое RectTransform и якоря?** **RectTransform** — это компонент, который есть у всех UI-элементов. Он определяет позицию, размер и **якоря (anchors)** элемента. **Якоря** — это точки привязки UI-элемента к родителю. Они определяют, как элемент будет позиционироваться и масштабироваться при изменении размера экрана. Например, якоря "Top Stretch" означают, что элемент растягивается по ширине и привязан к верхнему краю.

1. Выберите **Content** (внутри Scroll View).
2. У **Content** в **RectTransform** (компонент в Inspector) выставьте якоря как **Top Stretch** (растянуть по ширине, привязать к верху):
   - **Anchors**: Min **(0, 1)**, Max **(1, 1)** — это означает растяжение по ширине и привязку к верху.
   - **Pivot**: **(0.5, 1)** — точка поворота/масштабирования (центр по ширине, верх).
   - **Left** = 0, **Right** = 0, **Top** = 0 — отступы от краёв родителя.
   - **Height**: любое стартовое значение (например 200–500) — дальше высоту будет задавать Layout/Content Size Fitter.
3. На **Content** добавьте **Vertical Layout Group**:
   - **Child Control Width**: On
   - **Child Control Height**: On
   - **Child Force Expand Width**: On
   - **Child Force Expand Height**: Off
   - **Padding**: 5 (или по вкусу)

4. Правый клик по **Content** → **UI** → **Text - TextMeshPro** (при первом TMP-элементе импортируйте TMP Essentials).
5. Назовите объект **Text_Log**.
6. Настройте **RectTransform** у **Text_Log**:
   - Anchor: **Top Stretch** (Min (0,1) / Max (1,1)), **Pivot (0.5,1)**.
   - Left/Right = 0 (ширина управляется родителем), Height можно оставить как есть — будет подстроено по тексту.
7. У **TextMeshPro - Text (UI)**:
   - **Font Size**: 18–22.
   - **Alignment**: по левому краю, по верху (**Left**, **Top**).
   - **Wrapping**: включено (**Enabled**).
   - **Overflow**: **Overflow** (чтобы текст мог выходить за пределы по высоте и прокручиваться).
   - **Text**: оставьте пустым (""). Текст будет добавляться автоматически через **UIManager** при запуске игры.
   
   **Важно:** Не задавайте начальный текст вручную — UIManager сам добавит приветственное сообщение при старте игры. Если вы уже задали текст, очистите поле **Text** в Inspector.

### 2.4. Авто-высота контента (Content Size Fitter)

1. Выберите **Content**.
2. Добавьте компонент **Content Size Fitter**:
   - **Horizontal Fit**: **Unconstrained**
   - **Vertical Fit**: **Preferred Size**

**Важно:** `Content Size Fitter` на **Content** корректно работает именно вместе с **Vertical Layout Group** (он считает “preferred height” `Content` по дочерним элементам через layout-систему). После этого длинный текст в **Text_Log** будет увеличивать высоту **Content**, и скролл будет работать.

## 3. Поле ввода (InputField) внизу

### 3.1. Создание InputField

1. Правый клик по **Canvas** → **UI** → **Input Field - TextMeshPro** (или обычный **Input Field**; для единообразия с логом лучше TMP).
2. Назовите объект **InputField_Command**.
3. Настройте **RectTransform**:
   - **Anchor**: нижний центр (Bottom-Center), Min (0.5, 0), Max (0.5, 0).
   - **Pivot**: (0.5, 0).
   - **Pos X**: 0, **Pos Y**: 30, **Width**: 600, **Height**: 40.
   
   **Примечание:** При фиксированных якорях (Min X = Max X, Min Y = Max Y) Unity показывает **Pos X**, **Pos Y**, **Width**, **Height**. Если нужна привязка к краям с отступами, используйте якоря с растяжением (например, Bottom-Stretch), тогда появятся поля **Left**, **Right**, **Top**, **Bottom**.

### 3.2. Настройка TMP Input Field

- **Placeholder**: текст-подсказка, например «Введите команду (attack, hit, stats)...».
- **Character Limit**: 0 или 100 — по желанию.
- **Content Type**: **Standard** или **Autocorrected**.
- **Line Type**: **Single Line**.
- **Caret Blink Rate**: 0.85.

**Примечание:** Позже (в разделе 6) мы подключим этот InputField к скрипту **UIManager**, чтобы он мог читать команды из поля ввода.

## 4. Панель статистики справа (HP, сила, ловкость)

### 4.1. Панель

1. Правый клик по **Canvas** → **UI** → **Panel** (или **Image**).
2. Назовите **Panel_Stats**.
3. **RectTransform**:
   - **Anchor**: правый верхний угол (Top-Right), Min (1, 1), Max (1, 1).
   - **Pivot**: (1, 1).
   - **Pos X**: -10, **Pos Y**: -10.
   - **Width**: 220, **Height**: 180.
   - Или **Left** = -230, **Right** = -10, **Top** = -10, **Bottom** = 10 (относительно anchor).

### 4.2. Заголовок и поля статов

1. Внутри **Panel_Stats** создайте **Text - TextMeshPro** с названием **Text_StatsTitle**, текст: «Статистика».
   
   **Настройка RectTransform для Text_StatsTitle:**
   - **Anchor**: Top-Stretch (Min (0, 1), Max (1, 1)) или Top-Left (Min (0, 1), Max (0, 1)).
   - **Pivot**: (0.5, 1) для центрирования или (0, 1) для выравнивания по левому краю.
   - Если Top-Stretch: **Left** = 5, **Right** = 5, **Top** = -5, **Height** = 30.
   - Если Top-Left: **Pos X** = 10, **Pos Y** = -10, **Width** = 200, **Height** = 30.
   - **Font Size**: 18–20, **Alignment**: Center (или Left), **Top**.

2. Под ним создайте ещё один **Text - TextMeshPro** — **Text_Stats**.
   - В нём будет одна или несколько строк: HP, сила, ловкость (значения подставит **UIManager**).
   - Пример начального текста: «HP: -\nСИЛ: -\nЛОВ: -».
   
   **Настройка RectTransform для Text_Stats:**
   - **Anchor**: Top-Stretch (Min (0, 1), Max (1, 1)) — чтобы текст растягивался по ширине панели.
   - **Pivot**: (0, 1) для выравнивания по левому краю сверху.
   - **Left** = 10, **Right** = 10, **Top** = -40 (или ниже, в зависимости от высоты заголовка), **Bottom** = 10.
   - **Font Size**: 16–18, **Alignment**: Left, **Top**.
   - **Wrapping**: Enabled (на случай длинных значений).

**Альтернативный способ:** Используйте **Vertical Layout Group** на **Panel_Stats**:
- Добавьте **Vertical Layout Group** на **Panel_Stats**.
- **Padding**: Left/Right = 10, Top/Bottom = 10.
- **Spacing**: 5–10.
- **Child Alignment**: Upper Left.
- **Control Child Size**: Width = On, Height = Off.
- **Child Force Expand**: Width = On, Height = Off.
- Тогда **Text_StatsTitle** и **Text_Stats** автоматически растянутся по ширине и расположатся вертикально.

**Примечание:** Позже (в разделе 6) мы подключим **Text_Stats** к скрипту **UIManager**, чтобы он мог обновлять значения HP, силы и ловкости.

## 5. Отправка команды по Enter

Есть два удобных способа.

### Вариант A: через UIManager (рекомендуется)

В **UIManager** в `Update()` проверять:

- если поле ввода активно и нажат **Enter** (или **Return**) — вызвать метод отправки команды (например `SubmitCommand()`), прочитать текст из InputField, обработать и очистить поле.

Тогда не нужно вешать отдельный скрипт на InputField.

### Вариант B: через событие Submit у Input Field

1. Выберите **InputField_Command**.
2. В компоненте **TMP_Input Field** (или **Input Field**) найдите блок **Events**.
3. В **On End Edit** добавьте вызов метода, например `UIManager.OnInputSubmit(string)`.
4. В скрипте **UIManager** в обработчике проверять: если аргумент не пустой или ключ — Enter (в TMP при Submit передаётся введённая строка). Тогда обрабатывать команду и очищать поле.

В данной инструкции используется **вариант A** (проверка Enter в **UIManager**), чтобы всё управление командами было в одном месте.

## 6. Скрипт UIManager.cs

### Что такое UIManager? (Объяснение для новичков)

**UIManager** — это скрипт (компонент), который управляет всеми элементами интерфейса вашей игры. Представьте его как "диспетчера" или "координатора" для UI.

#### Зачем он нужен?

Без UIManager каждый элемент UI (текстовое поле, кнопка, панель статистики) работал бы сам по себе, и вам пришлось бы писать код в разных местах. **UIManager** объединяет всё управление UI в одном месте, что делает код проще и понятнее.

#### Что делает UIManager в нашей текстовой RPG?

1. **Читает команды из поля ввода** — когда вы вводите "attack" и нажимаете Enter, UIManager получает этот текст.

2. **Выводит сообщения в лог** — после атаки или получения урона UIManager добавляет результат в большое текстовое поле (Text_Log), чтобы вы видели, что произошло.

3. **Обновляет статистику** — когда HP, сила или ловкость меняются, UIManager обновляет панель статистики справа, чтобы всегда показывать актуальные значения.

4. **Вызывает методы Warrior** — когда вы вводите команду "attack", UIManager вызывает метод `Warrior.Attack()`, который выполняет атаку и возвращает результат.

#### Как это работает простыми словами?

```
Вы вводите команду "attack" → UIManager получает текст → 
→ UIManager вызывает Warrior.Attack() → 
→ Warrior выполняет атаку и возвращает результат → 
→ UIManager добавляет результат в лог → 
→ UIManager обновляет статистику
```

#### Что такое Warrior?

**Warrior** — это скрипт (компонент), который представляет вашего игрового персонажа — воина. Он хранит характеристики (HP, сила, ловкость) и имеет методы для действий (атака, получение урона).

**Зачем нужна ссылка на Warrior в UIManager?** Чтобы UIManager мог вызывать методы Warrior (например, `Attack()` или `TakeDamage()`) когда вы вводите команды. Без ссылки UIManager не знал бы, где находится ваш персонаж и как с ним взаимодействовать.

---

### Что должен делать UIManager (техническое описание):

- Получать текст из поля ввода.
- Выводить сообщения в главное текстовое поле (Text_Log).
- Обновлять панель статистики (Text_Stats).
- По командам вызывать методы **Warrior** (Attack, TakeDamage и т.д.).

Скрипт **UIManager.cs** приведён в проекте в папке **Assets/Scripts/UIManager.cs**.

### Подключение в сцене

1. **Создайте пустой объект** (GameObject) для управления игрой:
   
   **Что такое GameObject?** **GameObject** — это любой объект в Unity (персонаж, кнопка, камера и т.д.). Это "контейнер" для **компонентов** — частей, которые добавляют функциональность (визуал, логика, физика и т.д.).
   
   **Что такое пустой объект?** Это GameObject без визуальных компонентов (без Mesh, Sprite и т.д.). Он используется как контейнер для скриптов и логики игры. В нашем случае он будет хранить компонент **UIManager**.
   
   **Способ 1 (рекомендуется):**
   - В панели **Hierarchy** правый клик → **Create Empty** (или **GameObject → Create Empty**).
   - Назовите объект **GameManager** (выделите его и нажмите F2, или дважды кликните на имя в Hierarchy).
   
   **Способ 2:**
   - Меню **GameObject** → **Create Empty**.
   
   **Альтернатива:** Можно использовать существующий объект **Canvas** — просто выберите его в Hierarchy.

2. **Добавьте компонент UIManager:**
   
   **Что такое компонент?** **Компонент (Component)** — это "часть" GameObject, которая добавляет ему функциональность. Например, компонент **Image** делает объект видимым, компонент **Button** делает его кнопкой. **Скрипт** (код на C#) — это тоже компонент, который добавляет логику. В нашем случае **UIManager** — это скрипт-компонент, который управляет UI.
   
   - Выберите созданный объект **GameManager** (или **Canvas**) в Hierarchy.
   - В панели **Inspector** нажмите кнопку **Add Component**.
   - В поле поиска (Search) введите **"UIManager"** — скрипт появится в результатах поиска.
   - Выберите **UIManager** из результатов — компонент добавится к объекту.
   
   **Примечание:** В Unity скрипты находятся через поиск по названию, а не через отдельную категорию "Scripts". Просто введите название скрипта в поиск.

3. **Создайте объект для Warrior** (если его ещё нет):
   - Создайте пустой объект так же, как в пункте 1 (правый клик в Hierarchy → **Create Empty**).
   - Назовите его **Warrior** (или **Player**).
   - Выберите этот объект и в **Inspector** нажмите **Add Component**.
   - В поле поиска введите **"Warrior"** — скрипт появится в результатах.
   - Выберите **Warrior** из результатов — компонент добавится к объекту.
   
   **Примечание:** Если объект с компонентом **Warrior** уже есть в сцене, этот шаг можно пропустить.

4. **Назначение ссылок в Inspector:**
   
   **Что такое ссылка?** **Ссылка (Reference)** — это "указатель" на другой объект. Когда вы перетаскиваете объект в поле в Inspector, вы создаёте ссылку на него, чтобы скрипт мог с ним работать. Без ссылок UIManager не знал бы, где находятся элементы UI и объект Warrior.
   
   Выберите объект с компонентом **UIManager** в Hierarchy. В Inspector вы увидите поля для ссылок:
   
   - **Main Log**: перетащите сюда объект **Text_Log** из Hierarchy (он находится внутри Scroll View → Viewport → Content).
   
   - **Input Field TMP**: перетащите сюда объект **InputField_Command** из Hierarchy (если используете TextMeshPro Input Field — рекомендуется).
   
   - **Input Field Legacy**: оставьте пустым или перетащите сюда **InputField_Command**, если используете обычный UI Input Field (не TMP).
   
   - **Stats Text**: перетащите сюда объект **Text_Stats** из Hierarchy (он находится внутри Panel_Stats).
   
   - **Warrior**: перетащите сюда объект с компонентом **Warrior** из Hierarchy.
   
   **Как перетащить:** Просто перетащите объект из панели **Hierarchy** в соответствующее поле в Inspector. Unity автоматически определит нужный компонент (TMP_Text, TMP_InputField, Warrior и т.д.).

После этого ввод команды и нажатие Enter будут обрабатываться **UIManager**, результат атаки/урона и статы будут выводиться в лог и в панель статистики.

## 7. Рекомендуемый макет (чтобы лог был главным)

По вашему скриншоту лог и поле ввода сейчас слишком маленькие. Ниже — удобный “макет по умолчанию” для текстовой RPG (черновик, но читаемый).

### 7.1. Панель лога (Panel_Log) — большая, слева/по центру

1. Выберите **Panel_Log**.
2. В **RectTransform** выберите якоря **Stretch-Stretch** (растянуть по экрану).
3. Задайте отступы (поля будут называться Left/Right/Top/Bottom):
   - **Left** = 10
   - **Top** = 10
   - **Bottom** = 80  *(место под поле ввода)*
   - **Right** = 260  *(место под панель статистики справа: 220 + отступы)*

После этого ваш Scroll View внутри Panel_Log тоже лучше растянуть (Stretch-Stretch) с отступами 10.

### 7.2. Поле ввода (InputField_Command) — шире и выше

1. Выберите **InputField_Command**.
2. В **RectTransform** выберите якоря **Bottom Stretch** (растянуть по ширине снизу).
3. Отступы:
   - **Left** = 10
   - **Right** = 260
   - **Pos Y** = 10  *(это расстояние от нижнего края при MinY=MaxY=0)*
   - **Height** = 55 (можно 50–60)

**Почему так?** При якорях **Bottom Stretch** по вертикали элемент **не растягивается** (MinY = MaxY = 0), поэтому Unity показывает **Pos Y** вместо поля **Bottom**.

Так ввод будет удобнее: команду видно целиком, проще редактировать.

### 7.3. Панель статистики (Panel_Stats) — справа

Оставьте как есть (Top-Right). Рекомендация:
- **Width** = 220
- **Height** = 180–220
- Отступы от правого/верхнего края: 10

## 8. Как отобразить эмодзи в логе (вместо квадратиков)

Если в логе вместо символов/эмодзи (⚔, 🗡, 💥) отображаются квадратики, значит **текущий Font Asset TextMeshPro не содержит нужных глифов**.

Важно: TextMeshPro обычно работает через SDF‑атлас. **Цветные emoji‑шрифты** (например, `Segoe UI Emoji`, `Noto Color Emoji`) часто **не подходят** для прямой генерации SDF‑Font Asset (в итоге получаются квадраты). Для “псевдо‑эмодзи” лучше использовать **символьные шрифты** (например, `Segoe UI Symbol`) или **Sprite Asset**.

### Способ 1 (рекомендуется): Symbol‑шрифт + Fallback Font Asset

1. Откройте: **Window** → **TextMeshPro** → **Font Asset Creator**.
2. В поле **Source Font File** выберите шрифт **Segoe UI Symbol** (часто отображается как `SEGUISYM`).
3. В **Character Set** выберите **Custom Characters** и добавьте символы `⚔🗡` (и `💥`, если он поддерживается).
   - Если Unity “не даёт вставить” символы в поле, используйте файл:
     - В проект уже добавлен файл `Assets/TMP/tmpro_chars.txt`
     - В выпадающем **Character Set** выберите **Characters from File** (если доступно) и укажите этот файл.
4. Нажмите **Generate Font Atlas** → **Save** и сохраните Font Asset, например `SegoeUISymbol_Fallback`.
5. Добавьте этот Font Asset как fallback:
   - Откройте ваш основной Font Asset (который стоит в `Text_Log`)
   - В Inspector найдите **Fallback Font Assets**
   - Добавьте туда `SegoeUISymbol_Fallback`

Если после этого `⚔`/`🗡` стали отображаться, но `💥` всё ещё квадрат — значит выбранный шрифт не содержит `💥` (это нормально).

### Способ 2: Замена эмодзи на текстовые символы (временное решение)

Если настройка шрифта кажется сложной или `💥` не поддерживается, замените “эмодзи” на символы, которые точно есть в шрифте.

В файле **Warrior.cs** замените:
- `⚔` → `[SWORD]` или `*`
- `🗡` → `[DAGGER]` или `→`
- `💥` → `[BOOM]` или `!!` / `*` / `✦`

Или используйте простые символы:
- `⚔️` → `[+]` или `*`
- `🗡️` → `[>]` или `→`
- `💥` → `[!]` или `!!`

### Способ 3: Использование Sprite Asset для эмодзи (продвинутый)

Можно создать Sprite Asset с изображениями эмодзи, но это более сложный вариант для новичков.

**Рекомендация:** Начните со **Способа 1** (Symbol‑шрифт + fallback). Если нужны именно “настоящие цветные эмодзи” — тогда лучше делать **Sprite Asset**.
